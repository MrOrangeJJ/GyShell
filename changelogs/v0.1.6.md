# v0.1.6 Release Notes

### English

**Visual Update**

- **Introduced a new app icon**.

**Skill Tool Update**

- **Upgraded `create_skill` to `create_or_rewrite_skill`**.
- **If the target skill name already exists in GyShell's own skills directory, it now rewrites (overwrites) that existing skill instead of creating another duplicate skill file**.
- **The tool description now explicitly requires full-content replacement when modifying a skill**: provide the complete new skill content in `content` and replace the original content entirely.

**Experimental Feature Update**

- **Added a new experimental switch: `First-Turn Thinking Model`**.
- **When enabled, only the first `model_request` in each run uses the Thinking model**.
- **All later `model_request` steps in the same run automatically fall back to the Global model**.
- **This switch follows the same session-window lock behavior as other experimental flags** (locked while session is busy, released on `SESSION_READY`).

### 中文 (Chinese)

**视觉更新**

- **更新了新的应用图标（icon）**。

**Skill 工具更新**

- **将 `create_skill` 升级为 `create_or_rewrite_skill`**。
- **当目标 skill 名称在 GyShell 自有 skills 目录中已存在时，改为直接重写（覆盖）原有 skill，而不是再创建一个同名重复 skill 文件**。
- **工具描述已明确修改规则**：若要修改 skill，必须在 `content` 中提供完整的新内容，并整体覆盖原内容。

**试验性功能更新**

- **新增试验性开关：`首轮使用思考模型`**。
- **开启后，每次 run 的第一次 `model_request` 使用 Thinking 模型**。
- **同一次 run 中后续所有 `model_request` 会自动回退到 Global 模型**。
- **该开关沿用现有试验性开关的 session 窗口锁定语义**（session busy 期间锁定，`SESSION_READY` 后释放）。
